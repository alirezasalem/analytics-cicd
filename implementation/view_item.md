# view_item

Tracks when a user views a product detail page.

---

## Overview

The `view_item` event fires when a user lands on a product detail page (PDP). This event is foundational for understanding product interest, measuring funnel drop-off from browsing to cart, and powering remarketing audiences based on viewed products. It captures the product's identifying information, category, brand, and price at the moment of view.

---

## Trigger Rules

- The event fires **on page load** of any product detail page
- Page path must match the pattern `/products/*`
- The product data must be available in the data layer before the event fires
- Do **not** fire on category pages, search results, or quick-view modals—only on full product detail pages
- Do **not** fire multiple times if the user refreshes the page (standard page load behavior handles this)

## Business Rules & Edge Cases

1. **Clear ecommerce object first**: Always push `{ ecommerce: null }` before the event to prevent stale data from previous ecommerce events bleeding into this hit.

2. **Price formatting**: The `price` and `value` fields must be numbers (not strings). Do not include currency symbols or thousand separators. Use decimal points for cents (e.g., `299.99`, not `"$299.99"` or `"299,99"`).

3. **Currency consistency**: The `currency` value must match the actual currency of the displayed price. If the site supports multiple currencies, pull this dynamically from the page context.

4. **Category hierarchy**: Use forward slashes to separate category levels (e.g., `Electronics/Audio/Headphones`). GA4 will automatically parse up to 5 levels into `item_category`, `item_category2`, etc.

5. **GTM trigger configuration**: Configure the GTM trigger as a Page View trigger with a condition that the Page Path matches the regex `^/products/.*` (or equivalent URL pattern for your site structure).

6. **Single item array**: Even though only one product is viewed, the `items` array structure is required by GA4's ecommerce schema. Always wrap the product in an array.
