# view_item

Tracks when a user views a product detail page.

---

## Overview

The `view_item` event fires whenever a user lands on a product detail page (PDP). This event is critical for understanding product interest, measuring browse-to-purchase funnels, and powering remarketing audiences based on viewed products. It captures the full product context including pricing, brand, and category at the moment of view.

---

## Trigger Rules

- **The user navigates to any page with a URL path matching `/products/*`** (e.g., `/products/blue-running-shoes`, `/products/12345`)
- The event fires on **page load**, not on user interaction
- The product data must be available in the data layer before the event fires
- **Do NOT fire** this event on category listing pages, search results, or cart pages—only on individual product detail pages
- For single-page applications (SPAs), fire on virtual page load when the product detail view renders, not just on initial hard page load

## Business Rules & Edge Cases

1. **GTM Trigger Configuration**: Use a Page View trigger type with a Page Path condition set to `matches RegEx` for `/products/.*` — do not use a Click trigger.

2. **Price Formatting**: Always pass `price` and `value` as numbers (not strings). Use decimal notation without currency symbols (e.g., `189.99`, not `"$189.99"`).

3. **Currency Consistency**: The `currency` value must match the currency in which `price` and `value` are expressed. If the user switches currency on the site, ensure the data layer reflects the updated currency.

4. **Category Hierarchy**: Use forward slashes to denote category depth (e.g., `"Footwear/Running/Performance"`). Limit to 5 levels maximum per GA4 recommendations.

5. **SPA Navigation**: For React, Vue, or other SPA frameworks, ensure this event fires on route change to a product page, not only on hard page refresh. Clear previous ecommerce data before pushing new view_item events to avoid data bleed.

6. **Custom Dimensions**: The following values are mapped to custom dimensions in GA4 for enhanced reporting:
   - `cd_product_id` ← `item_id`
   - `cd_product_brand` ← `item_brand`
   - `cd_product_category` ← `item_category`
