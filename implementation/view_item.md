# view_item

Tracks when a user views a product detail page.

---

## Overview

The `view_item` event fires whenever a user lands on a product detail page (PDP). This event is foundational for understanding product interest, measuring funnel drop-off from browse to cart, and powering remarketing audiences. It captures key product attributes to enable analysis by brand, category, and price point.

---

## Trigger Rules

- **Must fire:** On page load of any product detail page (URL path matches `/products/*`)
- **User action:** None required — this is a passive page view event, not a click
- **State preconditions:** Product data must be available in the data layer at page load time
- **Must NOT fire:** On category pages, search results, cart pages, or any non-PDP page
- **Must NOT fire:** If product data fails to load or is unavailable

## Business Rules & Edge Cases

1. **Clear ecommerce object first:** Always push `{ ecommerce: null }` before the event to prevent stale data from previous pages carrying over (critical for SPAs and back-button navigation).

2. **Price formatting:** The `price` and `value` fields must be numbers, not strings. Do not include currency symbols or thousand separators (use `189.00`, not `"$189.00"` or `"1,189.00"`).

3. **Currency consistency:** The `currency` value must match across all three locations (top-level, `ecommerce.currency`, and be appropriate for the `price`/`value` amounts).

4. **Single-product views:** For a standard PDP, the `items` array contains exactly one product. The `value` field should equal that item's `price`.

5. **GTM trigger configuration:** Configure the GTM tag to fire on Page View triggers with a path condition matching `/products/*` — do not use a Custom Event trigger for this implementation.

6. **Data availability timing:** Ensure product data is populated in the data layer before the GTM container snippet executes, or use a dataLayer.push() callback pattern if data loads asynchronously.
