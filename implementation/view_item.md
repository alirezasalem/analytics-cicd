# view_item

Tracks when a user views a product detail page.

---

## Overview

The `view_item` event fires when a user lands on a product detail page (PDP). This event is critical for understanding product interest, measuring the effectiveness of merchandising efforts, and building remarketing audiences based on product views. It also feeds into GA4's ecommerce funnel reports for view-to-purchase conversion analysis.

---

## Trigger Rules

- **Must fire** when a product detail page finishes loading (page load, not click)
- **Page path condition:** The URL must match `/products/*` (e.g., `/products/blue-running-shoes`)
- **Must NOT fire** on category listing pages, search results, or quick-view modals
- **SPA consideration:** If using a single-page application, fire this event on virtual page loads when the user navigates to a new product detail page
- **All product data must be available** in the data layer before the event fires

## Business Rules & Edge Cases

1. **GTM Trigger Configuration:** Use a Page View trigger with a page path condition matching `/products/*` to ensure the event only fires on product detail pages.

2. **Price formatting:** The `price` and `value` fields must be numbers (not strings). Do not include currency symbols, commas, or formatting—use `189.99` not `"$189.99"`.

3. **Currency consistency:** The `currency` value must match the currency of the `price` and `value` fields. If the user switches currencies on-site, ensure all three values reflect the same currency.

4. **Custom dimensions:** GA4 will receive three custom dimensions (`cd_product_id`, `cd_product_brand`, `cd_product_category`) for enhanced product-level reporting outside of standard ecommerce reports.

5. **Single item per event:** Unlike cart or purchase events, `view_item` should contain exactly one product in the `items` array—the product currently being viewed.

6. **Deduplication:** No explicit deduplication is defined. If the same product page reloads (browser refresh), the event will fire again. This is expected behavior for page load triggers.
