# view_item

Tracks when a user views a product detail page, capturing the product information for ecommerce analytics.

---

## Overview

The `view_item` event fires whenever a user lands on a product detail page (PDP). This event is essential for understanding product interest, measuring the top of the purchase funnel, and enabling remarketing audiences based on products viewed. It provides the foundation for calculating view-to-cart and view-to-purchase conversion rates.

---

## Trigger Rules

- ✅ The user navigates to a product detail page (URL matches `/products/*`)
- ✅ The page has fully loaded (fire on page load, not DOM ready)
- ✅ Product data is available in the data layer at the time of page load
- ❌ Do NOT fire on category pages, search results, or quick-view modals
- ❌ Do NOT fire multiple times for the same page view (e.g., on SPA route changes that don't represent a new product view)

## Business Rules & Edge Cases

1. **Clear ecommerce object first**: Always push `{ ecommerce: null }` before the event to prevent data from previous ecommerce events from persisting.

2. **Price formatting**: The `price` and `value` fields must be numeric decimals (e.g., `299.99`), not strings. Do not include currency symbols or thousand separators.

3. **Category hierarchy**: Use forward slashes to denote category hierarchy (e.g., `Electronics/Audio/Headphones`). GA4 will automatically parse up to 5 levels.

4. **SPA handling**: For single-page applications, ensure this event fires on virtual page loads when the user navigates to a new product. Bind to your router's navigation events, not just initial page load.

5. **GTM implementation**: Configure a Page View trigger with a Page Path condition matching `/products/*` regex pattern.

6. **Custom dimensions**: The following custom dimensions are mapped in GA4: `cd_product_id`, `cd_product_brand`, `cd_product_category`. Ensure these are registered in your GA4 property settings.
