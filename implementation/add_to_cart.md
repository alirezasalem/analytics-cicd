# add_to_cart

Fires when a user successfully adds a product to their shopping cart from the Product Detail Page.

---

## Overview

This event tracks successful cart additions on the Product Detail Page (PDP). It fires **only after the cart API confirms the item was added**, not on the initial button click. This distinction ensures accurate cart addition counts and prevents inflated metrics from failed or abandoned add attempts. The data supports conversion funnel analysis, product popularity reporting, and merchandising decisions.

---

## Trigger Rules

- ✅ User clicks the "Add to Cart" button on a Product Detail Page
- ✅ The cart API returns a **successful response** confirming the item was added
- ✅ Product data (ID, name, price, quantity) is available and valid
- ❌ Do **NOT** fire on button click alone — wait for API success
- ❌ Do **NOT** fire if the API returns an error or timeout
- ❌ Do **NOT** fire for "Save for Later" or wishlist additions

## Business Rules & Edge Cases

1. **Fire on API success only** — Attach the dataLayer push to the cart API success callback, not the click event handler. This ensures the event reflects actual cart state changes.

2. **GTM Trigger Configuration** — Use a Custom Event trigger in GTM listening for the `add_to_cart` event name. Do not use Click triggers.

3. **Value calculation** — The `value` parameter should equal `price × quantity` for the item being added. If adding multiple units, calculate accordingly (e.g., 2 items at $89.95 = value of $179.90).

4. **Currency consistency** — Currency must match the user's displayed price currency. Use ISO 4217 codes (USD, EUR, GBP). Never mix currencies within a single push.

5. **Price formatting** — Use decimal numbers without currency symbols or thousand separators. Correct: `89.95`. Incorrect: `"$89.95"` or `"89,95"`.

6. **Quantity as integer** — Always pass quantity as a whole number, even if your system supports fractional quantities for weighted goods.

7. **SPA considerations** — On single-page applications, ensure the ecommerce object is cleared before pushing new data if multiple add-to-cart events can occur without page reload. Consider pushing `{ ecommerce: null }` before each event if stale data is a concern.
