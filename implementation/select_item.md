# select_item

Tracks when a user clicks on a product card to view or select a specific item from a product listing.

## Overview

The `select_item` event fires whenever a user clicks on a product card anywhere on the site. This event captures which products attract user attention from listing pages, search results, recommendation widgets, and category grids. It provides critical data for understanding product discovery patterns, optimizing merchandising placements, and measuring the effectiveness of different product list contexts.

## Trigger Rules

- User clicks on a product card container element (not just the image or text, but the entire clickable card area)
- The click must occur on any product card across the entire site, including:
  - Category listing pages
  - Search results pages
  - Homepage recommendation carousels
  - Cross-sell/upsell widgets on product detail pages
  - Recently viewed sections
- The product card must have associated product data available in the data layer
- The `list_name` parameter must accurately reflect the page context and widget type where the click occurred
- Do NOT fire on clicks to non-product elements within a card (e.g., "Add to Wishlist" buttons should have their own events)

## Business Rules & Edge Cases

1. **List name consistency**: The `list_name` must follow a consistent naming convention combining page context and widget type (e.g., `"PDP - You May Also Like"`, `"Search Results - Main Grid"`, `"Category - Women's Tops"`). Coordinate naming with the analytics team.

2. **Position tracking**: `list_position` must be 1-based (first item = 1, not 0) and reflect the visual position the user sees, accounting for any lazy-loaded or paginated content.

3. **Currency hardcoding**: The spec indicates `currency` is hardcoded to `"EUR"`. If the site supports multiple currencies, this should be updated to pull dynamically from the page context.

4. **Click element targeting**: Configure GTM trigger to target the product card container element, not inner children. This ensures clicks anywhere on the card (image, title, price) all fire the event exactly once.

5. **SPA considerations**: On single-page applications, ensure the `list_name` updates correctly when the user navigates between different list contexts without a full page reload.

6. **Price formatting**: Send `price` as a raw number (e.g., `149.99`) without currency symbols, thousands separators, or string formatting.
