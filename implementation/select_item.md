# select_item

Tracks when a user clicks on a product card to view more details or navigate to the product page.

## Overview

The `select_item` event fires when a user clicks on a product card anywhere on the site. This event captures which products attract user interest from various listing contexts (search results, category pages, recommendation widgets, etc.). Understanding product selection patterns helps the business optimize product placement, merchandising strategies, and recommendation algorithms.

## Trigger Rules

- User clicks on a product card container element
- The click occurs on any product card site-wide, regardless of page type or widget
- The product must have identifiable product data available (product_id, product_name, price)
- The `list_name` parameter must reflect the current page context and widget type where the product card is displayed
- Do NOT fire on hover, focus, or keyboard navigation without a deliberate selection action
- Do NOT fire on clicks to secondary elements within the card (e.g., wishlist icon, quick-view modal trigger) unless they navigate to the product

## Business Rules & Edge Cases

1. **List name must reflect context**: The `list_name` value must accurately describe both the page type and widget type (e.g., "Homepage - Recently Viewed", "Search Results - Main Grid", "PDP - You May Also Like"). Do not use generic values like "product list".

2. **Position is 1-based**: The `list_position` starts at 1 for the first item in any list, not 0.

3. **Currency is hardcoded**: Currency is fixed to `"EUR"` regardless of user locale or display currency.

4. **GTM Click Element targeting**: In GTM, configure the trigger to target product card container elements specifically. Ensure the trigger selector is broad enough to capture clicks on child elements (images, titles, price) within the card.

5. **Price format**: The `price` value must be a numeric type (not a string), without currency symbols or thousand separators.

6. **Category hierarchy**: Use forward slashes to separate category levels (e.g., "Apparel/Tops/T-Shirts"), maintaining consistent hierarchy depth where possible.
