# select_item

Tracks when a user clicks on a product card to select an item from a product listing.

## Overview

The `select_item` event fires when a user clicks on any product card across the site to view more details or interact with that product. This event is critical for understanding which products capture user attention within different listing contexts (search results, category pages, recommendation widgets, etc.) and helps identify high-performing product placements and optimize merchandising strategies.

## Trigger Rules

- **Must fire when:** User clicks on a product card container element anywhere on the site
- **User action:** Click interaction on a product card (not hover, not keyboard navigation unless it results in selection)
- **Click target:** The product card container element, not individual elements within the card (image, title, price)
- **Context requirement:** The `list_name` parameter must accurately reflect the page context and widget type where the product appears (e.g., "Search Results", "Category: Electronics", "Homepage Recommendations")
- **Must NOT fire on:** Clicks on non-product elements within listings (pagination, filters, sort controls)
- **Must NOT fire on:** Product detail page interactions (only on listing/grid views)

## Business Rules & Edge Cases

1. **List name must reflect context:** The `list_name` value should clearly identify both the page type and widget type. Examples: `"Search Results"`, `"Category: Men's Footwear"`, `"Homepage: Recently Viewed"`, `"PDP: You May Also Like"`.

2. **Position tracking accuracy:** The `list_position` must reflect the visual position in the rendered list (1-based), accounting for any client-side filtering or sorting. If products are lazy-loaded, position should be calculated based on the full logical list, not just visible items.

3. **Currency is fixed to EUR:** Per the spec, currency is hardcoded to `"EUR"`. If multi-currency support is added later, this should be dynamically populated.

4. **GTM trigger configuration:** Use a Click Element trigger in GTM targeting product card container elements. Avoid triggering on child elements to prevent duplicate fires.

5. **SPA navigation:** On single-page applications, ensure the dataLayer is not cleared between virtual page views so that `select_item` events retain proper context.

6. **Price formatting:** Always pass `price` as a numeric type (e.g., `89.99`), never as a formatted string (e.g., `"€89.99"` or `"89,99"`).
