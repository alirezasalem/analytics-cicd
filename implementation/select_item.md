# select_item

Tracks when a user clicks on a product card to view more details or navigate to the product page.

---

## Overview

The `select_item` event fires whenever a user clicks on a product card anywhere on the site. This event captures which products attract user attention and from which list context (category page, search results, homepage carousel, or recommendations). It helps the business understand product discovery patterns and measure the effectiveness of different product listing placements.

---

## Trigger Rules

- **User Action:** The user clicks on a product card container element
- **Applicable Contexts:** Must fire on product cards across ALL of the following surfaces:
  - Category/collection pages
  - Search results pages
  - Homepage carousels
  - Recommendation widgets (e.g., "You may also like", "Recently viewed")
- **Click Target:** The trigger should target the entire clickable product card container, not just the image or title
- **Must NOT fire:** On non-product card elements, wishlist/favorite buttons within the card, or quick-add-to-cart buttons (those have separate events)

## Business Rules & Edge Cases

1. **List Position Indexing:** `list_position` must be 1-indexed (first item = 1, not 0). Ensure frontend pagination is accounted for—if showing page 2 with 20 items per page, the first item on page 2 is position 21.

2. **List Name Standardization:** Use consistent `list_name` values across the site. Recommended values:
   - `"Category Page"` — for collection/category listings
   - `"Search Results"` — for search result pages
   - `"Homepage Carousel"` — for featured products on homepage
   - `"Recommendations"` — for algorithmic recommendation widgets

3. **Currency Hardcoded:** The spec defines `currency` as `EUR`. If the site supports multiple currencies, this should be dynamically populated based on user locale/selection.

4. **Price Format:** `price` must be a numeric value without currency symbols or thousand separators. Use a period as the decimal separator (e.g., `149.99`, not `"€149,99"`).

5. **Category Path Delimiter:** Use forward slashes (`/`) to separate category hierarchy levels. Do not include leading or trailing slashes.

6. **GTM Implementation:** Use a Click Element trigger in GTM. Configure the trigger to match all clickable product card containers using a CSS selector that captures cards across all listing contexts. Verify the trigger fires correctly on dynamically loaded content (infinite scroll, AJAX-loaded carousels).
