# view_cart

Tracks when a user views their shopping cart page, capturing the cart contents and total value for ecommerce analysis.

## Overview

The `view_cart` event fires when a user navigates to the cart page and the cart contains at least one item. This event is critical for understanding cart engagement, identifying drop-off points in the purchase funnel, and enabling cart abandonment remarketing audiences. It captures the full cart state including all items, applied coupons, and total value.

## Trigger Rules

- The user must navigate to or land on the `/cart` page
- The cart must contain at least one item (do not fire for empty carts)
- The page must be fully loaded (fire on DOM ready or page load, not before)
- A sessionStorage flag `cart_viewed_{session_id}` must be checked to prevent duplicate fires on SPA re-renders within the same session
- If the flag exists for the current session, do NOT fire the event again
- After firing, set the sessionStorage flag to prevent duplicates

## Business Rules & Edge Cases

1. **Deduplication via sessionStorage**: Before firing, check for the existence of `cart_viewed_{session_id}` in sessionStorage. If present, suppress the event. After firing, write this flag immediately to prevent duplicate fires during SPA navigation or page re-renders.

2. **Empty cart handling**: Do not fire `view_cart` if the cart contains zero items. The trigger condition requires at least one item to be present.

3. **SPA re-render protection**: In single-page applications, users may navigate away from and back to the cart without a full page reload. The sessionStorage flag ensures only one `view_cart` event fires per session, regardless of navigation patterns.

4. **Session ID dependency**: The deduplication mechanism relies on a valid `session_id` value. Ensure the session ID is available before constructing the flag key.

5. **Coupon code optional**: If no coupon is applied, omit the `coupon_code` parameter entirely rather than sending an empty string or null value.

6. **Currency consistency**: The `currency` value must match the currency used for `cart_value` and all `items[].price` values. Do not mix currencies within a single event.
