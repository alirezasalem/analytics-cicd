Feature Name: Add to Cart
Event Name: add_to_cart

What should be tracked:
When a user successfully adds a product to their cart, we want to track this action 
for GA4 reporting. The event should only fire after the cart has confirmed the item 
was added — not just when the button is clicked.

Where it happens:
The "Add to Cart" button on product pages.

Data we need to capture (for GA4 ecommerce reports):
- The currency used (e.g. EUR)
- The total value of what was added
- Product details:
    - Product ID
    - Product name
    - Brand
    - Category
    - Price
    - Quantity added

Why we need this:
GA4 uses this data to populate ecommerce reports — add-to-cart rate, 
product performance, and funnel analysis (add to cart → checkout → purchase).

Important notes:
- Do NOT fire if the add-to-cart fails (e.g. out of stock, API error)
- If the user adds 2 units, the value should reflect the total (price × quantity)
