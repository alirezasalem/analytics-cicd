## Background

We need to track when a user submits their payment details during checkout. This is a standard GA4 ecommerce funnel event that sits between `begin_checkout` and `purchase`.


## Feature description

On the payment step of checkout, the user selects or enters a payment method and clicks the continue/confirm button. At that point the `add_payment_info` event fires with the cart contents and selected payment type.


## Business questions

- What payment methods are most commonly used?
- What is the drop-off rate between begin_checkout and add_payment_info?
- Do coupon codes correlate with specific payment types?


## Events to track

### Event: add_payment_info

**When it fires:** On click of the payment confirmation/continue button on the payment step of checkout.

**Data to capture:**
- `ecommerce.currency` — ISO 4217 currency code (e.g. `EUR`)
- `ecommerce.value` — total cart value
- `ecommerce.payment_type` — selected payment method (e.g. `Credit Card`, `PayPal`)
- `ecommerce.coupon` — coupon code if applied, empty string if not
- `ecommerce.items` — array of cart items, each with `item_id`, `item_name`, `price`, `quantity`

**Priority:** P1

**Edge cases:** Clear previous ecommerce object before push (`dataLayer.push({ ecommerce: null })`). Do not capture raw card numbers or any PII.

---


## Scope

**In scope:**
- Web checkout flow

**Out of scope:**
- Mobile app
- Guest checkout variations (same event, no special handling needed)


## Risks

| Risk | Notes |
|------|-------|
| PII exposure | Never capture card number, expiry, or CVV |
| Duplicate fires | Ensure event fires once per button click, not on page load |
