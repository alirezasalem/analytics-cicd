Feature Name: Product Detail Page View
Event Name: view_item

What should be tracked:
When a user lands on a product detail page (PDP), we want to track this for GA4
ecommerce reporting. The event fires automatically when the page loads — no
interaction needed.

Where it happens:
Any product detail page (e.g. /products/sample-product).

Data we need to capture (for GA4 ecommerce reports):
- Currency (e.g. EUR)
- Value of the product being viewed (the product price)
- Product details:
    - Product ID
    - Product name
    - Brand
    - Category
    - Price

Why we need this:
GA4 uses this event to populate ecommerce reports — product detail views,
funnel analysis (view → add to cart → purchase), and item-level performance.

Important notes:
- Event fires on page load, not on any click
- No user interaction required to trigger it
