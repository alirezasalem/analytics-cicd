# FB-005 — Select Item

## Summary
When a user clicks on a product card anywhere on the site (category pages, search results, homepage carousels, recommendations), we need to track which product they selected and where they selected it from.

## Business goal
Understand which products and which placements drive the most product detail page visits. Feed into click-through rate reporting per list type.

## Scope
- Web only
- All product card clicks site-wide
- Includes: category pages, search results, homepage carousels, recommendation widgets
- Excludes: clicks inside the product detail page itself

## Event details
- Event name: select_item
- Trigger: user clicks a product card (the whole card is clickable)
- Must capture: product ID, product name, price, brand, category, the list name the product appeared in (e.g. "Category - Running Shoes"), and the position of the product in that list (1-based index)
- Currency: EUR

## Notes
- The list name should reflect the page and widget context (e.g. "Homepage - Recommendations", "Search Results")
- Position index is 1-based (first product = 1)
- Fire on click, not on navigation completion
