# purchase

Tracks a completed transaction when a customer successfully places an order.

---

## Overview

The `purchase` event fires when a customer lands on the order confirmation page after completing checkout. This is the most critical conversion event for revenue attribution, allowing the business to measure total sales, average order value, shipping and tax revenue, coupon effectiveness, and product-level performance. It powers ROAS calculations and end-to-end funnel analysis.

---

## Trigger Rules

- **Must fire on**: Page load of `/checkout/confirmation`
- **User must have**: Completed a successful transaction (order confirmation data must be available in the data layer)
- **Deduplication required**: Each `transaction_id` must only be sent to GA4 once—enable transaction deduplication in the GA4 tag to prevent duplicate revenue on page refreshes or back-button navigation
- **Must NOT fire**: On checkout error pages, payment failure screens, or any page other than the confirmation page
- **Must NOT fire**: If `transaction_id` is missing or empty

## Business Rules & Edge Cases

1. **Transaction deduplication**: Enable transaction ID deduplication in the GTM GA4 tag configuration. Users may refresh the confirmation page or navigate back to it, and each `transaction_id` must only count once in GA4 reports.

2. **Items array requirements**: Every item in the `items` array must include `item_id`, `item_name`, `price`, `quantity`, and `item_category`. Missing required item fields will cause product-level reporting to fail silently.

3. **Price is unit price**: The `price` field in each item should be the per-unit price, not the line total. GA4 calculates line totals by multiplying `price × quantity`.

4. **Value should equal order total**: The `value` parameter should represent the final amount charged to the customer, including tax and shipping. Ensure this matches your backend order total.

5. **Currency consistency**: All monetary values (`value`, `tax`, `shipping`, item `price`) must be in the same currency. The `currency` code must be a valid ISO 4217 three-letter code.

6. **Coupon at order vs item level**: Use the top-level `coupon` field for order-wide discounts. If different coupons apply to individual items, add a `coupon` property to each item object instead.

7. **Empty coupon handling**: If no coupon was used, either omit the `coupon` field entirely or pass an empty string. Do not pass `null` or `undefined`.

8. **SPA consideration**: If your checkout flow is a single-page application, ensure this event fires only once per successful order, triggered by the order confirmation state—not by route changes alone.
