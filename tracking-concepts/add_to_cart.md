# add_to_cart

Tracks when a user successfully adds a product to their shopping cart.

## Overview

This event fires after a product has been confirmed added to the cart via a successful API response. It captures product details, pricing, and cart value to measure shopping behavior and conversion funnel performance. The event can fire from multiple surfaces across the site (product detail pages, category listings, quick-add modals, etc.), making it critical for understanding which touchpoints drive cart additions.

## Trigger Rules

- The user initiates an "add to cart" action (button click, quantity update, etc.)
- The cart API returns a successful response (HTTP 2xx status code)
- The event fires **after** API success confirmation, not on the initial click
- Must clear any previous ecommerce data before pushing the new event
- Implement debounce logic to prevent duplicate events from rapid clicks or network retries
- Do NOT fire if the API returns an error (4xx, 5xx) or times out
- Do NOT fire on page load if items are pre-populated in the cart from a previous session

## Business Rules & Edge Cases

1. **API-gated firing**: The event must only fire after receiving a 2xx response from the cart API. Bind the dataLayer push to the API success callback, not the click handler.

2. **Debounce duplicate events**: Implement a debounce mechanism (e.g., 500ms window or tracking the last item_id + timestamp) to prevent duplicate events from double-clicks or network retries.

3. **Clear ecommerce object first**: Always push `{ ecommerce: null }` before the event to prevent data bleed from previous ecommerce events in single-page applications.

4. **Multiple add surfaces**: This event fires from various UI surfaces (PDP, PLP quick-add, recommendations, cart page "add more"). The `item_list_id` and `item_list_name` should reflect where the user discovered the item, not where they clicked.

5. **Value calculation**: The `value` parameter should equal `price × quantity` for the items being added in this specific action, not the total cart value.

6. **Currency consistency**: Use the same currency code across all ecommerce events in a session. Format prices as numbers (not strings) without currency symbols.

7. **Empty string vs. null for optional fields**: Use empty strings (`""`) for optional string parameters when data is unavailable, rather than omitting the key or using null.

8. **Quantity updates**: If a user increases quantity of an existing cart item, fire add_to_cart with the incremental quantity added, not the new total quantity.
