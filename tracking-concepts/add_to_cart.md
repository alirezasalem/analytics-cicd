# add_to_cart

Tracks when a user successfully adds a product to their shopping cart.

---

## Overview

This event fires when a product is confirmed added to the cart, not merely when the "Add to Cart" button is clicked. It captures product details including ID, name, price, quantity, category, and variant to enable analysis of which products drive cart engagement. This data powers conversion funnel analysis and helps identify high-performing products.

---

## Trigger Rules

- ✅ User clicks the Add to Cart button on a product page (`/products/*`)
- ✅ The cart API returns a success response **OR** local cart state confirms the item was added
- ✅ Product data (ID, name, price, quantity) is available and valid
- ❌ Do **NOT** fire on the raw button click alone—wait for cart state validation
- ❌ Do **NOT** fire if the API returns an error or the cart update fails

## Business Rules & Edge Cases

1. **Deferred firing required**: The event must fire only after the cart API returns a success response or local cart state confirms the addition. Never fire on the button click itself.

2. **Value calculation**: The `ecommerce.value` field should reflect the total value added (unit price × quantity) if multiple units are added at once.

3. **Currency consistency**: Use ISO 4217 three-letter currency codes (e.g., `"USD"`, `"EUR"`, `"GBP"`). Ensure currency matches the price displayed to the user.

4. **Variant formatting**: Combine variant attributes into a single string if the product has multiple variant dimensions (e.g., `"Blue / Size 10"` rather than separate fields).

5. **SPA handling**: On single-page applications, ensure the ecommerce object is cleared before each push to prevent stale data from previous interactions bleeding into the new event.

6. **Quantity updates**: If the user adds a product already in their cart, this event should still fire with the incremental quantity added, not the new cart total.
