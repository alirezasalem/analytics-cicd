# add_to_cart

Tracks when a user successfully adds a product to their shopping cart from the product detail page.

---

## Overview

This event fires after a user clicks the "Add to Cart" button on a product detail page (PDP) **and** the cart API confirms the item was successfully added. It captures the product details and cart value to measure purchase intent and track ecommerce funnel performance. This event is critical for understanding which products drive engagement and for remarketing audiences.

---

## Trigger Rules

- ✅ User clicks the "Add to Cart" button on a product detail page
- ✅ The cart API returns a **successful response** confirming the item was added
- ✅ Product data (ID, name, price, quantity) is available from the page or API response
- ❌ Do **not** fire on button click alone — wait for API confirmation
- ❌ Do **not** fire if the API returns an error (out of stock, network failure, etc.)

## Business Rules & Edge Cases

1. **Fire only after API success**: The GTM trigger must listen for the `add_to_cart` custom event in the dataLayer, which should only be pushed after the cart API returns a successful response. Never fire on button click alone.

2. **Value calculation**: The `value` parameter should equal `price × quantity`. If a user adds 3 units at $249.99 each, `value` should be `749.97`.

3. **Currency consistency**: Both root-level `currency` and `ecommerce.currency` must match and use valid ISO 4217 codes (e.g., `"USD"`, `"EUR"`, `"GBP"`).

4. **Price format**: Use numeric values without currency symbols. Use decimals for cents (e.g., `249.99`, not `"$249.99"` or `24999`).

5. **Category hierarchy**: Use forward slashes to denote category levels (e.g., `"Apparel/Outerwear/Jackets"`), not arrays or pipes.

6. **SPA handling**: On single-page applications, ensure the dataLayer push occurs after virtual page state updates and cart API confirmation, not during optimistic UI updates.
