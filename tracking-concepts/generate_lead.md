# generate_lead

Tracks successful contact form submissions after server-side validation confirms the lead was captured.

---

## Overview

This event fires when a user successfully submits the contact form on the `/contact` page **and** the backend API returns a success response (HTTP 200 or 201). It captures metadata about which form generated the lead and its category for marketing attribution. This event is critical for measuring lead generation effectiveness and optimizing contact form performance.

---

## Trigger Rules

- User must be on the `/contact` page
- User must complete and submit the contact form
- The backend API must return a successful response (HTTP status 200 or 201)
- The event must fire **after** the API success response is received, not on form submit button click
- Do **not** fire this event if:
  - The API returns an error (4xx or 5xx status)
  - Form validation fails client-side
  - The user abandons the form without submitting
  - The same form submission triggers multiple API success responses (deduplicate)

## Business Rules & Edge Cases

1. **Fire only on API success**: This event must be triggered inside the success callback of the form submission API request. Do not fire on button click or client-side validation success alone.

2. **PII restriction (critical)**: Never capture user-entered content. Only push form metadata (`form_id`, `form_name`, `lead_category`). Do not include user name, email address, phone number, or message body in the dataLayer.

3. **Deduplication**: If your form submission logic could potentially receive multiple success responses (e.g., retry logic, network issues), implement a flag to ensure this event fires only once per user-initiated submission.

4. **SPA handling**: If the contact page is part of a single-page application, ensure the dataLayer push occurs after the virtual pageview and only when the API confirms success.

5. **Error state handling**: If the API returns an error after an initial success (rare edge case with eventual consistency), do not push a compensating event—handle this server-side.

6. **Lead category values**: Use consistent, lowercase, underscore-separated values for `lead_category` (e.g., `general_inquiry`, `sales_request`, `technical_support`). Coordinate with the marketing team to maintain a controlled vocabulary.
