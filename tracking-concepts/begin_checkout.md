# begin_checkout

Tracks when a user initiates the checkout process by landing on the checkout page.

---

## Overview

The `begin_checkout` event fires when a user arrives at the checkout page, signaling their intent to complete a purchase. This event is critical for measuring checkout funnel entry and identifying drop-off between cart and payment. It captures the cart value, any applied coupons, and the full list of items the user intends to buy.

---

## Trigger Rules

- ✅ The user lands on a page where the URL path matches `/checkout`
- ✅ The page has fully initialized and cart data is available in the data layer
- ✅ The user's cart contains at least one item (do not fire for empty carts)
- ✅ The ecommerce object must be cleared before pushing this event to prevent data bleed from previous ecommerce events
- ❌ Do NOT fire on checkout page refreshes if the event has already been sent for the same cart session
- ❌ Do NOT fire if the user is redirected away from checkout before the page fully loads

## Business Rules & Edge Cases

1. **Clear ecommerce object first**: Always push `{ ecommerce: null }` before the `begin_checkout` event to prevent stale data from previous ecommerce events (add_to_cart, view_item, etc.) from contaminating this hit.

2. **Cart items array structure**: Each object in the `items` array must follow GA4's item schema exactly. Do not nest items inside another wrapper object.

3. **Currency consistency**: The `currency` value must be a valid ISO 4217 code (e.g., `"USD"`, `"EUR"`, `"GBP"`). Ensure all `price` values in the items array are in the same currency.

4. **Value calculation**: The `value` parameter should equal the sum of all `(price × quantity)` for items in the cart, before taxes and shipping.

5. **Empty coupon handling**: If no coupon is applied, either omit the `coupon` parameter entirely or pass an empty string. Do not pass `null` or `undefined`.

6. **SPA checkout flows**: If your checkout is part of a single-page application, ensure this event fires on route change to `/checkout`, not just on hard page loads. Implement deduplication to prevent duplicate events on route re-entry.

7. **Deferred data availability**: If cart data is fetched asynchronously after page load, wait until the cart API response is received before firing this event. Do not fire with incomplete or placeholder data.
