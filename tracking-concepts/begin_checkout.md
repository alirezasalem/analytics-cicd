# begin_checkout

Tracks when a user initiates the checkout process from their shopping cart.

---

## Overview

The `begin_checkout` event fires when a user clicks the checkout button to start purchasing the items in their cart. This event is critical for understanding checkout funnel drop-off—it marks the boundary between browsing/cart behavior and committed purchase intent. By comparing this event volume to `purchase` completions, the business can identify friction points in the checkout flow.

---

## Trigger Rules

- The user clicks the checkout button on the cart page
- The cart must contain at least one item (cart is non-empty)
- The event fires **once per checkout initiation**—if the user navigates back to the cart and clicks checkout again, a new event fires
- Do NOT fire this event on page load of the checkout page; it must be tied to the explicit button click action
- Do NOT fire if the checkout button is disabled or if the click is programmatically blocked (e.g., validation errors preventing checkout)

## Business Rules & Edge Cases

1. **Currency format**: Always use uppercase ISO 4217 codes (e.g., `"USD"`, `"EUR"`, `"GBP"`). Do not include currency symbols in the `value` field—it must be a raw number.

2. **Value calculation**: The `value` should reflect the sum of `(price × quantity)` for all items. Do not include shipping, tax, or discounts unless those are already factored into your cart total at this stage.

3. **Items array integrity**: Every item in the `items` array must include `item_id`, `item_name`, `price`, and `quantity`. If any of these are missing for an item, the GA4 ecommerce reporting may be incomplete.

4. **Price precision**: Use decimal values for prices (e.g., `59.99`, not `60`). Avoid rounding errors by pulling values directly from your cart data source.

5. **SPA handling**: In single-page applications, ensure the dataLayer push happens on the click handler, not on route change. If checkout navigates to a new SPA view, bind the event to the button interaction before navigation occurs.

6. **Ecommerce object clearing**: Always push `{ ecommerce: null }` immediately before the `begin_checkout` push to prevent stale data from previous ecommerce events (like `view_item` or `add_to_cart`) from bleeding into this hit.
