# begin_checkout

Tracks when a user initiates the checkout process by landing on the checkout page.

## Overview

This event fires when a user arrives at the checkout page, signaling their intent to complete a purchase. It captures the cart contents, total value, and any applied discounts at the moment checkout begins. This data helps the business understand checkout funnel entry rates, average cart values at checkout initiation, and the effectiveness of coupon campaigns.

## Trigger Rules

- The user lands on a page with the path `/checkout`
- The event fires on **page load**, not on a button click or user interaction
- The cart must contain at least one item (the `items` array must be populated)
- The cart total and currency values must be available from the data layer before the event fires
- Clear any existing ecommerce data before pushing this event to prevent contamination from previous ecommerce events
- Do **not** fire this event on checkout page refreshes if the event has already been sent for this checkout session
- Do **not** fire if the user is redirected away from checkout before the page fully loads

## Business Rules & Edge Cases

1. **Ecommerce object clearing**: Always push `{ ecommerce: null }` immediately before the begin_checkout push to clear any residual ecommerce data from previous events (e.g., view_item, add_to_cart).

2. **Currency formatting**: The `currency` value must be a valid 3-letter ISO 4217 code (e.g., `USD`, `EUR`, `GBP`). Do not include currency symbols.

3. **Value calculation**: The `value` parameter should reflect the cart total after discounts but before tax and shipping are applied.

4. **Empty coupon handling**: If no coupon is applied, either omit the `coupon` parameter entirely or pass an empty string. Do not pass `null` or `undefined`.

5. **SPA navigation**: For single-page applications, ensure the event fires when the checkout route is activated via client-side routing, not just on hard page loads. Implement deduplication to prevent double-firing if the user navigates away and returns.

6. **Items array structure**: Each item in the array must include at minimum `item_id`, `item_name`, `price`, and `quantity`. Missing required fields will cause GA4 to reject the item from reporting.

7. **Price precision**: Use decimal numbers for `price` and `value` (e.g., `149.97`), not strings. Avoid floating-point precision issues by rounding to 2 decimal places.
