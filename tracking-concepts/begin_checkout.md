# begin_checkout

Tracks when a user initiates the checkout process by landing on the checkout page.

---

## Overview

The `begin_checkout` event fires when a user navigates to the checkout page, signaling their intent to complete a purchase. This is a critical funnel event that helps measure checkout abandonment rates and identifies where users drop off between cart and purchase. It captures the cart contents and total value at the moment checkout begins.

---

## Trigger Rules

- ✅ User lands on a page with path `/checkout`
- ✅ Event fires on **page load**, not on a button click or user interaction
- ✅ Cart must contain at least one item (the `items` array must be populated)
- ✅ Cart total value must be available at the time of page load
- ❌ Do NOT fire this event on other pages, even if they contain checkout-related content
- ❌ Do NOT fire if the user refreshes the checkout page (implement deduplication per session if needed)

## Business Rules & Edge Cases

1. **Page load timing**: Fire this event only after cart data is fully loaded and available. If cart data is fetched asynchronously, wait for the API response before pushing to dataLayer.

2. **SPA navigation**: For single-page applications, ensure this event fires on virtual page loads to `/checkout`, not just hard page refreshes. Hook into your router's navigation events.

3. **Currency format**: Use ISO 4217 three-letter currency codes (e.g., `"USD"`, `"EUR"`, `"GBP"`). Do not include currency symbols in the `value` field.

4. **Value calculation**: The `value` parameter should reflect the cart subtotal (sum of item prices × quantities). Do not include tax or shipping at this step unless they are already calculated and displayed.

5. **Empty cart redirect**: If a user navigates directly to `/checkout` with an empty cart, do NOT fire this event. Handle this case by redirecting users or showing an empty cart message.

6. **Session deduplication**: Consider implementing logic to prevent duplicate `begin_checkout` events if users refresh the page or navigate away and return within the same session.
