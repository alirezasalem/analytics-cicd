# scroll_depth

Tracks when a user scrolls past the 50% threshold of a page, measuring content engagement depth.

---

## Overview

This event fires when a visitor scrolls past 50% of the total page height, indicating meaningful engagement with the page content. It helps the business understand whether users are consuming content beyond the initial viewport, which is critical for evaluating content effectiveness, identifying high-engagement pages, and optimizing content placement for calls-to-action.

---

## Trigger Rules

- **User action:** The user scrolls the page and the scroll position crosses the 50% vertical threshold of the total page height
- **Threshold:** Fires once when scroll depth reaches or exceeds 50% of the document height
- **Deduplication:** Should fire only once per page view — if the user scrolls up and back down past 50%, do not fire again
- **SPA consideration:** On single-page applications, reset the scroll tracking state when the route/page changes
- **Must NOT fire:** On pages where content fits within the viewport (no scrollable content), or if the page height cannot be reliably calculated

## Business Rules & Edge Cases

1. **Single-fire per page view:** Implement a flag (e.g., `window._scrollDepth50Fired = true`) to ensure the event fires only once per page load, even if the user scrolls up and down multiple times.

2. **GTM implementation recommended:** This event is best implemented via Google Tag Manager's built-in Scroll Depth trigger with a 50% vertical threshold, rather than custom JavaScript. GTM handles the scroll calculation and deduplication automatically.

3. **SPA page transitions:** For single-page applications, reset the scroll tracking flag when the virtual page view fires or when `history.pushState` is called, allowing the event to fire again on the new "page."

4. **Short pages:** If the page content is shorter than the viewport height (no scroll required), this event will never fire — this is expected behavior, not an error.

5. **Dynamic content:** If page content loads asynchronously and changes the document height after initial render, ensure scroll percentage calculations account for the final page height, not the initial height.

6. **Mobile considerations:** On mobile devices with bounce/elastic scrolling, ensure the calculation uses actual scroll position relative to document height, ignoring overscroll effects.
