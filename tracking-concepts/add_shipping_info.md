# add_shipping_info

Tracks when a user successfully saves their shipping method selection during checkout.

---

## Overview

This event fires when a customer completes the shipping information step of the checkout flow and their selected shipping method is validated by the backend. It captures which shipping tier they chose (e.g., standard, express, overnight) along with the current cart value. This data helps the business understand shipping preference patterns and identify potential drop-off points between shipping selection and payment.

---

## Trigger Rules

- The user must be on the `/checkout/shipping` page
- The user clicks the "Continue to Payment" or "Save Shipping" button (exact selector TBD)
- The shipping validation API must return a **200 success response** before firing
- Do **not** fire on button click alone — wait for backend confirmation
- Do **not** fire if the API returns an error (4xx/5xx response)
- Do **not** capture any PII (addresses, names, phone numbers) — only the shipping tier string is permitted

## Business Rules & Edge Cases

1. **Fire on API success only**: The dataLayer push must occur inside the success callback of the shipping validation API call, not on the button click event. This ensures we only track confirmed shipping selections.

2. **PII restriction**: Do not include any shipping address fields, recipient names, phone numbers, or email addresses in the dataLayer push. Only the `shipping_tier` string is permitted.

3. **Coupon handling**: If no coupon is applied, omit the `coupon` parameter entirely rather than passing an empty string or null.

4. **Currency formatting**: Use ISO 4217 three-letter codes (e.g., `USD`, `EUR`, `GBP`). Do not include currency symbols.

5. **Value precision**: Use decimal numbers for `value` and `price` fields. Do not include currency symbols or thousand separators. Example: `149.97` not `$149.97` or `1,149.97`.

6. **SPA considerations**: If the checkout is a single-page application, ensure the ecommerce object is cleared (`{ ecommerce: null }`) before each push to prevent stale data from previous steps contaminating this event.

7. **Selector clarification needed**: The exact button selector for the shipping confirmation action requires confirmation from the development team before GTM implementation.
