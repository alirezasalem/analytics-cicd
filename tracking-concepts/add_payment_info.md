# add_payment_info

Tracks when a user successfully submits their payment information during checkout.

## Overview

This event fires when a customer completes the payment information step of the checkout flow. It captures which payment method was selected, the order value, and any applied coupons. This data helps the business understand payment method preferences, identify friction points in checkout, and measure the effectiveness of promotional coupons.

## Trigger Rules

- ✅ User clicks the payment submission button on `/checkout/payment`
- ✅ Payment form validation completes successfully (no validation errors)
- ✅ Event must fire **before** any redirect to external payment providers (PayPal, Stripe hosted pages, etc.)
- ✅ Cart must contain at least one item
- ❌ Do NOT fire on validation failure (invalid card number, expired card, etc.)
- ❌ Do NOT fire if the user navigates away without submitting
- ❌ Do NOT fire multiple times for retry attempts within the same checkout session

## Business Rules & Edge Cases

1. **Timing is critical**: The event MUST fire after payment validation succeeds but BEFORE any redirect to external payment gateways. For PayPal, Stripe Checkout, or similar hosted payment pages, push the event synchronously before initiating the redirect.

2. **Payment type values**: Use consistent, lowercase identifiers for `payment_type`. Common values: `"credit_card"`, `"debit_card"`, `"paypal"`, `"apple_pay"`, `"google_pay"`, `"klarna"`, `"afterpay"`, `"bank_transfer"`.

3. **Coupon handling**: If no coupon is applied, omit the `coupon` parameter entirely rather than passing an empty string or null.

4. **Currency consistency**: The `currency` value must match the currency used for `value` and all `items[].price` values. Do not mix currencies.

5. **Value calculation**: The `value` parameter should reflect the cart total at the moment of payment submission, including any discounts, taxes, and shipping costs that have been calculated.

6. **Selector clarification needed**: The payment submission button selector is not specified in the current spec. Coordinate with the front-end team to identify the correct selector (e.g., `#submit-payment-btn`, `[data-action="submit-payment"]`).
