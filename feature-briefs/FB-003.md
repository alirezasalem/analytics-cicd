# FB-003: Track Completed Purchases

## What & Why
When a customer completes a purchase, we need to capture the full transaction details. This data feeds our revenue reporting in GA4 and helps us understand coupon impact, shipping conversion, and new vs. returning customer split.

This event should fire once on the order confirmation page, after payment is confirmed.

## When does it fire?
User lands on `/checkout/confirmation` after a successful payment.

## What do we need to capture?

- Order ID (so we can tie GA4 data back to our backend)
- Total order value, tax, and shipping cost
- Currency
- Coupon code used (if any)
- Whether the customer is new or returning
- User ID (if logged in)
- All products in the order — name, ID, price, quantity, category

## Important notes for the dev team
- The order ID must be unique — if a user refreshes the confirmation page, the event should not fire twice
- Guest checkouts have no user ID — just leave it out entirely, don't send null or empty string
- Coupon field should be empty string if no coupon was used
- Customer type should be based on whether they have a prior completed order in our system, not just whether they have an account

## Out of scope
- Failed or pending payments
- Partial refunds
